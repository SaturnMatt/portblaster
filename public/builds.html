Builds

The project keeps three release targets and a separate validation folder:

  • pb20.exe: 20 KB target release build.
  • pb50.exe: 50 KB target release build.
  • pb100.exe: 100 KB target release build.
  • pbjelly\: jelly validation builds and pbjelly.exe.

All three release builds come from portblaster.c with compile-time feature flags.

  • pb20: minimal safe server.
  • pb50: log, metrics, Copy URL, and Browse root.
  • pb100: log, metrics, streaming file responses, tray behavior, optional status/config/listing/range checks, explicit bind=all LAN testing, and explicit ipv6=1 local serving.

The jelly utility checks normal pages, rejected paths, malformed request lines, unsupported methods, listener bind mode, and repeated request latency. Reports include a defense summary showing served, blocked, and unexpected probes.

The validation builds accept --start for automated local testing. The release builds stay manual and size-focused.
